\section{Methodology}
\label{sec:methodology}

The full rationale, including the pitfalls each control guards against, is in the
replication package (\texttt{METHODOLOGY.md}); we summarize here.

\paragraph{Factors.}
We vary three factors: \emph{access layer} (nine levels: \texttt{pg} and
\texttt{mysql2} native drivers, Knex query builder, and the Drizzle, Prisma,
Sequelize, TypeORM, Objection, and MikroORM ORMs); \emph{engine} (PostgreSQL~16,
MySQL~8.4); and \emph{access pattern}, realized as five Express endpoints:
point read by primary key, newest-first range scan, deep/nested fetch (post +
author + comments each with their author), per-author aggregation, and insert.
The native drivers are engine-specific; every other layer runs on both engines.

\paragraph{Workload and data.}
A minimal blog schema (\texttt{authors} 1--$*$ \texttt{posts} 1--$*$
\texttt{comments} $*$--1 \texttt{authors}) is loaded from a deterministic seed
(1{,}000 authors, 20{,}000 posts, 200{,}000 comments) by the native driver,
independently of any layer under test. The working set fits in memory.

\paragraph{Controls.}
The connection pool is fixed at ten for every adapter, so the comparison isolates
access-layer overhead rather than pool tuning. Every adapter returns an identical
result shape to Express and uses that layer's recommended idiomatic strategy to
avoid the N+1 problem on the deep-fetch endpoint (join, \texttt{include},
\texttt{with}, or eager loading), documented per adapter.

\paragraph{Measurement.}
Load is generated by autocannon~\cite{autocannon} in a process separate from the
server. Each (layer, engine, endpoint) cell is warmed (measurements discarded) to
absorb JIT compilation, pool fill, and plan-cache effects, then measured over
$\mathit{REPEATS}$ runs; we report the median. For every cell we record throughput
(requests/second) and latency percentiles p50, p90, p97.5, and p99. The engines
run with durability disabled and the working set resident in memory, so results
reflect access-layer cost rather than storage latency. The replication package
auto-captures the hardware/software environment, reported here once measured.
