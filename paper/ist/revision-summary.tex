\section*{Summary of Revisions}

\textbf{Manuscript:} A Comparability Protocol for Benchmarking Relational Database Access Layers in Express.js \textbf{Author:} Mateusz Miotk \textbf{Journal:} Information and Software Technology

This revision keeps the primary benchmark matrix unchanged and addresses the remaining conceptual, construct-validity, and artifact-auditability concerns.

\section*{Conceptual framing}

\begin{itemize}
  \item Added and distinguished CYNTHIA (ICSE 2021) as prior cross-ORM differential-equivalence work.
  \item Narrowed novelty to the integrated benchmark-admission protocol rather than correctness checking or differential testing themselves.
  \item Removed global "complete" and universal "necessary" claims; compliance levels now state the claims they license.
  \item Renamed RQ2 to backend-stack transfer and removed causal attribution to the DBMS alone.
  \item Audited same-SQL prose so the result remains a compound standardized contrast that isolates no mechanism.
\end{itemize}

\section*{Construct validity and experiments}

\begin{itemize}
  \item Archived the recorded pre-freeze Wayback responses for all nine selected treatments with capture timestamps, source URLs, SHA-256 hashes, evidence terms, a reproducible conflict rule, and an explicit capture-time limitation.
  \item Documented each canonical constructor operation.
  \item Added a standalone 30-block canonicalization benchmark and generated Supplement Table S44. Deep-fetch construction costs 9.32--9.40 microseconds, 0.078\% of the fastest corresponding HTTP p50.
  \item Quantified two saturation-target sensitivities: bootstrap confidence bounds yield 48.6--51.7\% and 68.0--72.4\%; replacing the target with the full-sweep maximum yields 46.8--51.2\% and 65.5--71.6\% (Supplement Table S45).
  \item Restricted the matched-utilization headline to stable 50\% and most 70\% cells; 85--95\% results are sensitivity trends.
\end{itemize}
